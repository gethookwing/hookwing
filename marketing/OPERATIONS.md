# Marketing Operations

## TinaCMS content workflow

## Purpose

Create and review blog/docs content using Tina with Git and Cloudflare preview checks.

## Responsibilities

- Draft/edit content in Tina (`/admin`) or markdown files in `website/content/`.
- Confirm titles, summaries, and publish intent (`draft` state).
- Request preview review on Cloudflare `design-lab` URL before merge.

## Editorial flow

1. Start/update a content branch.
2. Edit content and metadata.
3. Open PR with clear change summary.
4. Review on preview URL and request fixes if needed.
5. Approve merge when content quality and accuracy are confirmed.

## Quality bar

- Clear headline and summary.
- No unresolved placeholders.
- Links and media paths are valid.
- No confidential information included.
