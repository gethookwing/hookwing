Hookwing API Documentation

Everything you need to receive, route, and deliver webhooks reliably. Base URL: https://api.hookwing.com

Getting Started

Get your first webhook delivered in three steps.

1. Create an account

curl -X POST https://api.hookwing.com/v1/auth/signup \
  -H "Content-Type: application/json" \
  -d '{
    "email": "you@example.com",
    "password": "your-secure-password"
  }'

Response includes your workspace info and an API key (hk_live_...). Save the key — it's shown only once.

2. Create a webhook endpoint

curl -X POST https://api.hookwing.com/v1/endpoints \
  -H "Authorization: Bearer hk_live_yourkey" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://your-app.com/webhooks",
    "description": "Production webhook receiver"
  }'

You'll receive an endpoint ID (ep_...) and a signing secret (whsec_...). Store the signing secret securely.

3. Send a webhook

curl -X POST https://api.hookwing.com/v1/ingest/ep_yourEndpointId \
  -H "Content-Type: application/json" \
  -d '{
    "event_type": "order.created",
    "payload": {"order_id": "ord_123", "amount": 4999}
  }'

Hookwing receives the event, signs it with HMAC-SHA256, and delivers it to your endpoint URL with automatic retries.

Authentication

All authenticated endpoints require a Bearer token in the Authorization header:

Authorization: Bearer hk_live_yourApiKey

API keys are scoped to a workspace. You can create multiple keys with different permissions via the dashboard or API.

Public endpoints: POST /v1/ingest/:endpointId, GET /health, GET /tiers, GET /openapi.json, GET /api/pricing, GET /api/status, and POST /v1/auth/signup do not require authentication.

Rate Limits

Rate limits are per-workspace and tier-gated. Every response includes rate limit headers:

HeaderDescription
X-RateLimit-LimitMaximum requests per second for your tier
X-RateLimit-RemainingRequests remaining in the current window
X-RateLimit-ResetUnix timestamp when the window resets

When exceeded, the API returns 429 Too Many Requests with a Retry-After header.

Webhook Signatures

Every webhook delivery is signed with HMAC-SHA256 using the endpoint's signing secret. Verify signatures to ensure events are authentic.

Hookwing sends these headers with each delivery:

HeaderDescription
hookwing-signatureHMAC-SHA256 hex digest
hookwing-timestampUnix timestamp (seconds) of the delivery

To verify:

  1. Concatenate timestamp.body (the timestamp, a dot, and the raw request body)
  2. Compute HMAC-SHA256 with your whsec_... signing secret
  3. Compare the result to the hookwing-signature header (constant-time comparison)
  4. Reject if the timestamp is more than 5 minutes old (replay protection)
// Verification in Node.js
const crypto = require('crypto');

function verifyWebhook(body, signature, timestamp, secret) {
  const payload = `${timestamp}.${body}`;
  const expected = crypto
    .createHmac('sha256', secret)
    .update(payload)
    .digest('hex');
  return crypto.timingSafeEqual(
    Buffer.from(signature),
    Buffer.from(expected)
  );
}

SDK

The @hookwing/sdk npm package provides a typed client with built-in signature verification.

npm install @hookwing/sdk
import { Webhook } from '@hookwing/sdk';

const wh = new Webhook('whsec_yourSigningSecret');

// In your webhook handler:
app.post('/webhooks', (req, res) => {
  const isValid = wh.verify(
    req.body,                          // raw body string
    req.headers['hookwing-signature'],
    req.headers['hookwing-timestamp']
  );

  if (!isValid) {
    return res.status(401).send('Invalid signature');
  }

  const event = JSON.parse(req.body);
  // Process the event...
  res.status(200).send('ok');
});

The SDK uses crypto.subtle for cross-runtime compatibility (Node.js, Deno, Cloudflare Workers, Bun).

CLI

The @hookwing/cli provides command-line access to the full API.

npm install -g @hookwing/cli
# Authenticate
hookwing auth login

# Manage endpoints
hookwing endpoints list
hookwing endpoints create --url https://your-app.com/webhooks

# View events
hookwing events list --limit 10

# Check deliveries
hookwing deliveries list --status failed

Configuration is stored at ~/.hookwing/config.json.

Errors

All errors return a consistent JSON format:

{
  "error": "Human-readable error message",
  "status": 400
}
StatusMeaning
400Bad request — invalid input or missing required fields
401Unauthorized — missing or invalid API key
403Forbidden — tier limit reached (e.g. max endpoints)
404Not found — resource doesn't exist
409Conflict — duplicate resource (e.g. email already registered)
429Rate limited — slow down, check Retry-After header
500Internal server error — contact support
